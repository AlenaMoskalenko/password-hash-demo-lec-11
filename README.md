# password-hash-demo-lec-11
Поле passwordHash у БД — це не зашифрований пароль і не рядок, який треба “розкодувати”; це результат роботи password-hashing алгоритму, призначений саме для перевірки автентичності.

Для bcrypt перевірка виконується через bcrypt.compare(plainPassword, hash), а для Argon2 — через argon2.verify(hash, plainPassword); обидві функції повертають булевий результат збігу.

Рядок хешу зазвичай уже містить службову інформацію, потрібну для перевірки, зокрема параметри алгоритму, а в Argon2 — ще й PHC-формат на кшталт $argon2id$v=19$m=...,t=...,p=...$....

Як з ним працювати
На реєстрації бери plainPassword, обчислюй hash, а в БД зберігай тільки email та passwordHash.

На логіні діставай користувача з БД, передавай введений пароль і збережений хеш у compare() або verify(), і лише при true відкривай сесію або видавай JWT.

Ніколи не роби логіку типу “дістань пароль із БД і порівняй рядки”, бо в правильній схемі відкритого пароля в БД взагалі не існує.
