Password Hash Demo
bcrypt + argon2id

Демо правильної реєстрації та логіну без збереження відкритого пароля

Ця сторінка показує потік: сервер приймає пароль, хешує його, кладе в БД лише хеш, а під час логіну робить тільки перевірку через compare() або verify().

password → hash DB stores hash only JWT after success

Потік автентифікації

1
Реєстрація
Сервер отримує пароль, обчислює bcrypt/Argon2id хеш і записує в БД тільки його.
2
Логін
Сервер дістає хеш з БД і виконує перевірку введеного пароля через compare/verify.
3
Успіх
Лише після вдалого збігу сервер видає access/refresh token або відкриває сесію.

Інтерактивне пояснення

Готово до запуску демонстрації.
Сервер не відновлює пароль із хешу. Він тільки перевіряє, чи введений пароль дає такий самий результат при верифікації.
У БДЩе нічого не збережено
ПорівнянняОчікує запуску
ТокениНе видані

Node.js код

bcryptреєстрація + логін
import bcrypt from 'bcrypt';

async function registerUser(email, plainPassword) {
  const passwordHash = await bcrypt.hash(plainPassword, 12);
  await users.insertOne({ email, passwordHash });
}

async function loginUser(email, plainPassword) {
  const user = await users.findOne({ email });
  if (!user) throw new Error('Invalid credentials');

  const ok = await bcrypt.compare(plainPassword, user.passwordHash);
  if (!ok) throw new Error('Invalid credentials');

  return issueTokens({ sub: user._id, email: user.email });
}
argon2Argon2id за замовчуванням
import argon2 from 'argon2';

async function registerUser(email, plainPassword) {
  const passwordHash = await argon2.hash(plainPassword, {
    type: argon2.argon2id,
    memoryCost: 19456,
    timeCost: 2,
    parallelism: 1,
  });

  await users.insertOne({ email, passwordHash });
}

async function loginUser(email, plainPassword) {
  const user = await users.findOne({ email });
  if (!user) throw new Error('Invalid credentials');

  const ok = await argon2.verify(user.passwordHash, plainPassword);
  if (!ok) throw new Error('Invalid credentials');

  return issueTokens({ sub: user._id, email: user.email });
}

Що важливо пам’ятати

A
Не зберігайте plaintext
У БД має бути тільки passwordHash, а не password.
B
Не "розшифровуйте" пароль
Пароль не має повертатися у відновлюваному вигляді ні API, ні адміністратору.
C
Видавайте токени лише після verify
JWT, session або refresh token створюються тільки після успішної перевірки хешу.
